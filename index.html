ip命令

发表于 2025-03-11 更新于 2025-03-22 分类于 linux ， 系统分析工具

1.  基本用法

ip link  # 查看网络设备运行状态。 ip -s link  # 查看网络设备运行状态， 包含数据收发的详细信息。 ip addr  # 查看网络接口信息。 ip addr show wlp4s0 # 查看指定设备的信息， 例如我的无限网卡wlp4s0。 ip route  # 查看路由信息。 ip addr add #增加ip地址。

2.  命令输出解释

2.1 ip link

可以看到， 我的电脑上有4张网卡： lo: 本地回环。 eno1: 一个板载的以太网端口， en表示以太网。 enp3s0: 一个通过PCI总线连接的以太网端口，p表示PCI，3, s, 0应该也是硬件总线相关的。 wlp4s0:  一个无线网卡。 NO-CARRIER： 表明它也没有建立物理连接。 LOWER_UP： 标志，这意味着它已成功与一个无线接入点建立了连接。 在虚拟机中网卡一般为eth0, 因为找不到相关的信息来命名。

2.2  ip route

第一行： default: 表示这是一条默认路由。当一个数据包的目的地不在其他更具体的路由表中时，将使用此路由。 via 192.168.251.245: 这是网关的IP地址，所有非本地的数据包都将通过这个IP地址进行转发。 dev wlp4s0: 数据将通过名为 wlp4s0 的网络接口发送。通常，这种命名方式表示这是一个无线网络接口。 proto dhcp: 路由是通过DHCP协议自动获取的。 src 192.168.251.39: 当有数据需要发送到外部网络时，将会使用这个源IP地址。 metric 600: 路由的优先级度量值，数字越小表示优先级越高。 第二行： 192.168.251.0/24: 这是一个子网定义，表示所有目标为 192.168.251.X (X的范围是0到255)的流量都属于这个子网。 dev wlp4s0: 指定数据将通过名为 wlp4s0 的网络接口发送。 proto kernel: 表明这条路由是由内核根据接口配置自动生成的。 scope link: 表示这个路由适用于直接连接到同一链路的设备。 src 192.168.251.39: 发往该子网的数据包将使用这个源地址。 metric 600: 同样代表了这条路由的优先级度量值。

3.  给指定网口添加/删除一个IP

sudo ip addr add 192.168.251.101/24 dev eth0 sudo ip addr del 192.168.251.101/24 dev eth0 这都是临时修改， 重启后就会失效。

00 IOStream 概述

发表于 2025-03-10 更新于 2025-03-22 分类于 cpp ， 流

1.  IOStream

IO可以分为流式I/O, 记录I/O, C++的IOStream采用了流式I/O.

2.   流式I/O的两个问题

2.1.  表示形式

使用“格式化”或“解析”在“数据的内部表示”与“字符序列”之间进行转换。

  • 格式化: 将数据的内部表示（二进制表示）转换为字符序列。
  • 解析: 将字符序列转换为数据的内部表示。

也就是说， cout(输出流)对应的是格式化， cin(输入流)对应解析。

2.2.  与外部设备的通信

所谓外部设备，就是文件、终端、内存等。不同的外部设备，访问的逻辑不同，C++的I/O库进行了封装，对上层提供了统一的接口来访问外部设备。

3.   输入输出涉及的操作

3.1  输出流

格式化 –> 缓存 –> 编码转换 –> 输出 缓 存：缓冲区， 才会执行下一步. 编码转换：UTF-8, GBK.

3.2  输入流

读取 –> 编码转换 –> 解析 –> 解析 解析：将字符序列转换为数据的内部表示。

4.  IOStream的实现原理

C++使用模板封装字符特性， 使用继承封装设备特性。例如：std::ifstream就是std::basic_ifstream的别名。

virtualBox网络配置

发表于 2025-03-09 更新于 2025-03-22 分类于 virtualization ， VirtualBox

1. 相关网络配置概念

1.1. 网关

两个网络之间的关口在家庭网络环境中，无线路由器通常也充当网关的角色，简单来说，网关就是一个网络通往另一个网络的入口。

1.2. DHCP

简单说就是给局域网中的设备分配IP的， DHCP（Dynamic Host Configuration Protocol，动态主机配置协议），是一种网络管理协议，用于自动化地分配IP地址和相关网络配置给网络中的设备。当一个设备连接到一个使用DHCP的网络时，会发送一条广播消息请求IP地址配置信息。网络上的DHCP服务器收到这个请求后，会从预先定义好的IP地址池中分配一个未使用的IP地址、子网掩码、默认网关以及DNS服务器地址等必要的网络配置信息给该设备。

1.3. 路由器

在小型网络结构中， 例如家里面的无线路由器， 就负责网关和DHCP的工作。

1.4. 关于NAT

网络地址转换，就是两个网络之间的地址的转换，一般由于路由器，或者防火墙等负责。

2. VirtualBox网络配置

2.1. 主机模式 Host-Only

创建一块Host-Only虚拟网卡， 就是创建一个封闭网络，虚拟网卡为每一个虚拟机分配一个局域网IP，该局域网内部，虚拟机之间可以通信，虚拟机和宿主机可以通讯，但是不能访问互联网。

2.2. 网络地址转换(NAT)

这个在VirtualBox虚拟机管理器中是默认的，这种模式下，VirtualBox会为每一个虚拟机创建一个网关/DHCP，所有虚拟机的IP都是相同的， 默认是10.0.2.15。虚拟机可以访问互联网，可以访问宿主机，但是反过来不行，虚拟机之间也不能相互访问。

2.3. NAT 网络

这种模式下，VirtualBox创建了一个交换机(10.0.2.0)供所有的虚拟机使用，并且分配了一个网关(10.0.2.1)，一个DHCP服务器(10.0.2.3)，和默认的NAT相比， 这种配置允许虚拟机之间通信。
