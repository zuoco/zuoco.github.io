1. 同步的概念

C++流默认使用缓冲区机制，数据不会立即输出或写入文件，而是保存在缓存区中，积累到一定量后一次性写入。

2. 输出流的同步。

  1. 输出流的同步函数：
  1. 操纵符std::flush刷新缓存区;
  2. 操纵符std::endl回车并刷新缓存区。

3. 输入流的同步

实现逻辑由编译器决定。

4. unitbuf

一个操纵符，只要大于一个字符，就会自动刷新缓存区。

5. 基于绑定的同步

流可以绑定到一个输出流上，这样每次输入、输出前都会先刷新输出流的缓存区。
可以有多个流绑定到一个输出流上，例如A、B绑定到输出流C上，但是反过来是不行的。
A –> C
B –> C
例如cin绑定到cout，这样就可以在输入时刷新cout缓存区，保证在读取前将提示信息打印出来。

6. 与C标准库同步

当前代码中既有C++，也有C时，或者调用的库中有C的输出输出代码，C++的输入输出默认与C的同步，可以通过sync_with_stdio关闭同步。

1. IOStream

IO可以分为流式I/O, 记录I/O, C++的IOStream采用了流式I/O.

1.1. 流式I/O的两个问题

1.2. 表示形式

使用格式化或解析在数据的内部表示与字符序列之间进行转换。
格式化: 是将数据的内部表示（二进制表示）转换为字符序列，对于同一块内存，以不同的类型格式化，格出来的结果自然不同。
解析: 是将字符序列转换为数据的内部表示，也就是说，cout(输出流)对应的是格式化，cin(输入流)对应解析。

1.3. 与外部设备的通信

所谓外部设备，就是文件、终端、内存等。不同的外部设备，访问的逻辑不同，
C++的I/O库进行了封装，对上层提供了统一的接口来访问外部设备。

2. 输入输出涉及的操作

2.1. 输出流

格式化 –> 缓存 –> 编码转换 –> 字符序列。

格式化：二进制表示到字符序列。
缓存：积累到一定量后，一次性输出。
编码转换：UTF-8, GBK。

2.2. 输入流

读取 –> 编码转换 –> 缓存 –> 解析 –> 内部表示。
