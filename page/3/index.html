1. 相关网络配置概念

虚拟机会创建虚拟网卡， 虚拟网卡和物理机网卡是同等的， 是两个网段。
比如说我的物理机连接家里的无线网络, 网络拓扑如下：

1.1. 网关

两个网络之间的关口
在家庭网络环境中，无线路由器通常也充当网关的角色，
简单来说，网关就是一个网络通往另一个网络的入口。

1.2. DHCP

给局域网中的设备分配IP的
DHCP（Dynamic Host Configuration Protocol，动态主机配置协议），
是一种网络管理协议，用于自动化地分配IP地址和相关网络配置给网络中的设备。
当一个设备连接到一个使用DHCP的网络时，会发送一条广播消息请求IP地址配置信息。
网络上的DHCP服务器收到这个请求后，会从预先定义好的IP地址池中分配一个未使用的IP地址、子网掩码、默认网关以及DNS服务器地址等必要的网络配置信息给该设备。

1.3. 路由器

在小型网络结构中， 例如家里面的无线路由器， 就负责网关和DHCP的工作。

1.4. 关于NAT

网络地址转换，就是两个网络之间的地址的转换，一般由于路由器，或者防火墙等负责。

2. VirtualBox网络配置

2.1. 主机模式 Host-Only

创建一块Host-Only虚拟网卡， 就是创建一个封闭网络，
虚拟网卡为每一个虚拟机分配一个局域网IP，
该局域网内部，虚拟机之间可以通信，虚拟机和宿主机可以通讯，但是不能访问互联网。
仅限黑框内可以相互访问。

2.2. 网络地址转换(NAT)

这个在VirtualBox虚拟机管理器中是默认的，
这种模式下，VirtualBox会为每一个虚拟机创建一个网关/DHCP，
所有虚拟机的IP都是相同的， 默认是10.0.2.15。
虚拟机可以访问互联网，可以访问宿主机，但是反过来不行，虚拟机之间也不能相互访问。

2.3. NAT 网络

这种模式下，VirtualBox创建了一个交换机(10.0.2.0)供所有的虚拟机使用。
并且分配了一个网关(10.0.2.1)，一个DHCP服务器(10.0.2.3)，
和默认的NAT相比， 这种配置允许虚拟机之间通信。
